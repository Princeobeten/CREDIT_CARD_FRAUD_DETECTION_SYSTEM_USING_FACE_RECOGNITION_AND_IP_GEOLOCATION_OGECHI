## 1. Critical breaking changes / things NOT to do from memory

1. **`params` and `searchParams` are `Promise`s — you must `await` them.** [BREAKING] Applies to pages, layouts, **and** dynamic Route Handlers (`await ctx.params`). The old synchronous `{ params }: { params: { id: string } }` destructure is wrong everywhere.
2. **Middleware is renamed to "Proxy".** [BREAKING] The file is **`proxy.ts`** (root or `src/`), and you export a function named **`proxy`** (named or `default`). `middleware.ts` / `export function middleware` no longer the convention. **One proxy file per project.**
3. **`cookies()` and `headers()` from `next/headers` are async** — `const c = await cookies()`. [BREAKING] Synchronous calls are invalid.
4. **`fetch()` is NOT cached by default; Route Handlers are NOT cached by default.** [BREAKING] Inverse of 12–14. Caching is opt-in via the **`'use cache'`** directive. Only `GET` handlers can opt into static caching (`export const dynamic = 'force-static'`); `POST/PUT/PATCH/DELETE` are never cached.
5. **`'use cache'` cannot sit directly in a Route Handler body** — extract into a helper function, pair with `cacheLife()` from `next/cache`.
6. **`refresh()` is now exported from `next/cache`** (not `router.refresh()`) for refreshing the client router after a mutation. `updateTag`/`revalidateTag` are distinct (they revalidate *tagged data*; `refresh()` does not). [BREAKING]
7. **`useActionState` is from `react`** (not `react-dom`) and returns a **3-tuple** `[state, action, pending]`. `useFormStatus` is from `react-dom`. The old `useFormState` is gone. [BREAKING]
8. **Tailwind v4 setup**: single `@import "tailwindcss";` line + the `@tailwindcss/postcss` PostCSS plugin. **No** `tailwind.config.js` required, **no** `@tailwind base/components/utilities` triple. [BREAKING] Do not scaffold the v3 setup. (This project is already on v4 — see §8.)
9. **Auto-generated global type helpers** `PageProps<'/route'>`, `LayoutProps<'/route'>`, and `RouteContext<'/route/[id]'>` exist with no import. Generated by `next dev` / `next build` / **`next typegen`**. Stop hand-rolling `{ params: { id: string } }`. [BREAKING]
10. **`export const runtime = 'edge'` is not supported with Cache Components** — Node.js runtime only; Proxy also runs on Node.js by default now (not forced Edge). [BREAKING]
11. **Don't do auth checks in layouts** — layouts don't re-render on client navigation (Partial Rendering). Check in pages, the DAL, Server Actions, and Route Handlers.
12. **A page-level redirect does NOT protect Server Actions** defined on that page — each action is an independent POST endpoint. Re-verify auth + role *inside every action*.
13. **Never trust Proxy alone for authorization.** Proxy = coarse optimistic gate (cookie decrypt only). Real role checks happen at the data layer.
14. **Never hit MongoDB/Mongoose from Proxy** — Proxy runs on every route, including prefetches. Mongoose also wants Node runtime. Keep DB in the DAL / Server Components / Route Handlers.
15. **Unprefixed env vars become `""` (empty string) in the client bundle, not `undefined`.** Only `NEXT_PUBLIC_*` reach the client, and they are **inlined and frozen at build time**. Never `NEXT_PUBLIC_` a secret (Mongo URI, JWT secret).
16. **Cannot define a Server Action inside a Client Component.** Define in a `'use server'` file, import it. Webcam/face-api components are clients, so the action lives elsewhere.
17. **Component state persists across navigations** (routes hidden via React `<Activity>`, not unmounted) when Cache Components is on. Webcam streams won't tear down on navigate — clean up `MediaStream` tracks in an effect cleanup.

## 2. Route Handlers (API endpoints) — signatures + working example

- **File:** `src/app/**/route.ts`. A `route.ts` and a `page.tsx` **cannot** coexist at the same segment. `app/page.tsx` + `app/api/route.ts` is fine.
- **Methods:** `export async function GET/POST/PUT/PATCH/DELETE/HEAD/OPTIONS(request: Request)`. Unsupported methods auto-return **405**. If `OPTIONS` is undefined, Next adds it and sets `Allow` from your defined methods.
- **Async dynamic params** [BREAKING]: type the context with the global `RouteContext<'/path/[id]'>` and `await ctx.params`.
- **Body is read-once**: `request.json()` / `request.formData()` / `request.text()` throw on a second read — `request.clone()` first if you need it twice. `GET`/`HEAD` carry no body.
- **Prefer POST for sensitive data** (IP, geolocation, face descriptors) — `GET` URLs get cached/logged.

```ts
// src/app/api/transactions/[id]/route.ts
import { type NextRequest, NextResponse } from "next/server";
import { verifySession } from "@/lib/dal";
import { getTransaction } from "@/lib/transactions"; // server-only DAL

export async function GET(_req: NextRequest, ctx: RouteContext<"/api/transactions/[id]">) {
  const session = await verifySession();              // re-check auth at the data layer
  if (!session) return new Response(null, { status: 401 });
  if (!["analyst", "admin"].includes(session.role)) return new Response(null, { status: 403 });

  const { id } = await ctx.params;                    // params is a Promise — await it
  const tx = await getTransaction(id);
  if (!tx) return NextResponse.json({ error: "Not found" }, { status: 404 });
  return NextResponse.json(tx); // return a DTO, not the raw Mongoose doc
}
```

## 3. Auth & route protection (Proxy vs middleware) — cookies + roles

**Layered model:** (a) **Proxy** = optimistic cookie-only gate; (b) **DAL `verifySession()`** = real per-request check in Server Components/Actions/Handlers.

```ts
// session helpers (Jose) — src/lib/session.ts
import "server-only";
import { SignJWT, jwtVerify } from "jose";
import { cookies } from "next/headers";

const encodedKey = new TextEncoder().encode(process.env.SESSION_SECRET); // openssl rand -base64 32

export async function encrypt(payload: { userId: string; role: "cardholder" | "analyst" | "admin"; expiresAt: Date }) {
  return new SignJWT(payload).setProtectedHeader({ alg: "HS256" }).setIssuedAt().setExpirationTime("7d").sign(encodedKey);
}
export async function decrypt(session = "") {
  try { return (await jwtVerify(session, encodedKey, { algorithms: ["HS256"] })).payload; }
  catch { return undefined; }
}
export async function createSession(userId: string, role: "cardholder" | "analyst" | "admin") {
  const expiresAt = new Date(Date.now() + 7 * 24 * 60 * 60 * 1000);
  const session = await encrypt({ userId, role, expiresAt });
  (await cookies()).set("session", session, {       // cookies() is ASYNC
    httpOnly: true, secure: true, sameSite: "lax", path: "/", expires: expiresAt,
  });
}
export async function deleteSession() { (await cookies()).delete("session"); }
```

```ts
// proxy.ts (project root or src/) — OPTIMISTIC, cookie only, NO DB
import { NextRequest, NextResponse } from "next/server";
import { decrypt } from "@/lib/session";

const protectedRoutes = ["/dashboard", "/transactions", "/admin"];
const publicRoutes = ["/login", "/signup", "/"];

export default async function proxy(req: NextRequest) {
  const path = req.nextUrl.pathname;
  const cookie = req.cookies.get("session")?.value; // or (await cookies()).get(...)
  const session = await decrypt(cookie);

  if (protectedRoutes.some((p) => path.startsWith(p)) && !session?.userId)
    return NextResponse.redirect(new URL("/login", req.nextUrl));
  if (publicRoutes.includes(path) && session?.userId && !path.startsWith("/dashboard"))
    return NextResponse.redirect(new URL("/dashboard", req.nextUrl));
  return NextResponse.next();
}

export const config = { matcher: ["/((?!api|_next/static|_next/image|.*\\.png$).*)"] };
```

## 4. Server vs Client Components + where webcam/face-api lives

- **Server Components are the default**, `async`, can `await` DB/`fetch` directly. Credentials stay server-side.
- **`'use client'`** goes at the **top of the file, above imports**. The boundary is **transitive**: every import and directly-rendered child joins the client bundle — never import `server-only`/Mongoose code downstream of it.
- **Interleaving exception:** a Server Component passed as `children`/props into a Client Component renders on the server and arrives as RSC output — the supported way to nest server content under a client wrapper.

**Webcam / face-api.js MUST be Client Components** (they use `getUserMedia`, `localStorage`, `<video>`, etc.):

```tsx
// src/app/(cardholder)/_components/FaceCapture.tsx
"use client";
import { useEffect, useRef } from "react";
import { submitFaceDescriptor } from "@/lib/actions/face"; // imported 'use server' action

export default function FaceCapture() {
  const videoRef = useRef<HTMLVideoElement>(null);
  useEffect(() => {
    let stream: MediaStream;
    (async () => {
      stream = await navigator.mediaDevices.getUserMedia({ video: true });
      if (videoRef.current) videoRef.current.srcObject = stream;
      // load face-api models, run detection...
    })();
    return () => stream?.getTracks().forEach((t) => t.stop()); // routes are hidden not unmounted — must stop tracks
  }, []);
  return <video ref={videoRef} autoPlay muted />;
}
```

Rules: wrap third-party client-only libs (face-api) in your own `'use client'` module if they lack the directive. Lazy-load face-api models and keep them off shared layouts so `<Link>` prefetch/hydration isn't blocked. Any prop passed to a client component must be serializable — pass a sanitized DTO, never a Mongoose document, function, or class. Guard server modules with `import "server-only"` (Mongoose, session, DAL, DTO). Use `experimental_taintUniqueValue` / `experimental_taintObjectReference` (enable `experimental.taint: true`) to keep secrets out of the client.

## 6. Caching defaults in Next 16

**Default = dynamic. Nothing is cached unless you opt in.** [BREAKING vs 12–14]

- **Opt in** with the **`'use cache'`** directive (top of an async helper, a component/page/layout body, or a file) plus `cacheLife(profile)` / `cacheTag(tag)` from `next/cache`. Args + closed-over values become part of the cache key (so passing `userId` gives per-user entries).
- **Stay uncached / fresh-per-request**: wrap the work in `<Suspense>` (streaming) or read runtime data (cookies/headers).
- **`fetch()` is uncached** and blocks render until done unless wrapped in `<Suspense>` or `'use cache'`. Wrap repeated user/role lookups in React `cache()` so they run once per request.

**Cache Components** (opt-in via `next.config.ts`): makes PPR the default; runtime data access (`await cookies()/headers()`, `searchParams`, `params`, `Math.random()`, `Date.now()`, DB reads) **must** be inside `<Suspense>` or it throws *"Uncached data was accessed outside of `<Suspense>`"* at dev/build.

```ts
// next.config.ts (NOT enabled yet in this project — enable deliberately)
import type { NextConfig } from "next";
const nextConfig: NextConfig = { cacheComponents: true };
export default nextConfig;
```

```tsx
// reading the JWT cookie is runtime data → must be inside Suspense (or its own uncached component)
import { Suspense } from "react";
export default function Dashboard() {
  return (<><h1>Dashboard</h1><Suspense fallback={<p>Loading…</p>}><RoleAwarePanel /></Suspense></>);
}
```

Migration of old segment configs (when Cache Components on): delete `export const dynamic = 'force-dynamic'` (already default); `export const revalidate = 3600` → `'use cache'` + `cacheLife('hours')`; `fetchCache` not needed; `force-static` → `'use cache'` + `cacheLife('max')` and remove runtime reads; `runtime = 'edge'` unsupported.

Serverless: default `'use cache'` is in-memory and does NOT persist across requests — use **`'use cache: remote'`** for durable shared caching of Mongoose/DB results. Per-cardholder fraud data is inherently dynamic — fine to leave uncached and stream via Suspense.

## 8. Env vars + Tailwind v4

**Env load order** (first match wins): `process.env` > `.env.$(NODE_ENV).local` > `.env.local` > `.env.$(NODE_ENV)` > `.env`. `.env.local` is **not** read when `NODE_ENV=test`.

- **`NEXT_PUBLIC_*` only** reach the client, **inlined and frozen at build time**. Reference them by **full static literal** (`process.env.NEXT_PUBLIC_X`) — dynamic lookups (`process.env[name]`) are `undefined` in the browser.
- **Server secrets** (`MONGODB_URI`, `SESSION_SECRET`, geo/IP API keys) stay unprefixed and server-only. Generate `SESSION_SECRET` with `openssl rand -base64 32`.
- To read an env var **at runtime** (not frozen at build), opt into dynamic rendering with `await connection()` from `next/server`.
- `@next/env`'s `loadEnvConfig(process.cwd())` loads `.env*` outside the Next runtime (Mongoose CLI/scripts/tests). `.env` supports `$VAR` references.

**Tailwind v4** (already configured in this project) — do NOT add a `tailwind.config.js` or v3 directives:

